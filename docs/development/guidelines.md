# Руководящие принципы разработки

Этот документ описывает стандарты и принципы разработки Oasis 3D.

## Стандарты кода

### Python

- Следуйте [PEP 8](https://www.python.org/dev/peps/pep-0008/)
- Используйте type hints где возможно
- Максимальная длина строки: 100 символов
- Используйте `black` для форматирования кода

### Документация

- Все публичные функции и классы должны иметь docstrings
- Используйте формат Google style docstrings
- Обновляйте документацию при изменении API

### Тестирование

- Покрытие тестами должно быть не менее 80%
- Пишите unit-тесты для всех новых функций
- Используйте `pytest` для тестирования

## Структура проекта

```
Oasis 3D/
├── src/              # Исходный код
├── tests/            # Тесты
├── docs/             # Документация
├── examples/         # Примеры
└── scripts/          # Вспомогательные скрипты
```

## Git Workflow

### Ветки

- `main` — стабильная версия
- `develop` — разработка
- `feature/*` — новые функции
- `bugfix/*` — исправления ошибок
- `hotfix/*` — срочные исправления

### Коммиты

Используйте понятные сообщения коммитов:

```
feat: добавить новую функцию X
fix: исправить ошибку Y
docs: обновить документацию
refactor: рефакторинг кода
test: добавить тесты
```

## Code Review

- Все изменения требуют ревью
- Минимум один одобряющий ревьюер
- Все проверки должны пройти успешно
